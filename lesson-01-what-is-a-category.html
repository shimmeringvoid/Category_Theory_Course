Intro to Category Theory · Lesson 01

What Is a Category?

Stop asking what mathematical objects are made of. Ask only how they map to one another — and almost everything turns out to live in the arrows.

§1 — Why arrows?

The same shape, over and over

Walk through any corner of mathematics and you keep meeting the same arrangement: a kind of thing, paired with a kind of structure-preserving map between those things.

Sets come with functions. Groups come with homomorphisms. Vector spaces come with linear maps. Topological spaces come with continuous functions. Data types come with programs. In every one of these worlds the maps compose — do one, then another — composition is associative, and every object has a do-nothing map that changes nothing.

Category theory takes that recurring shape and makes it the object of study. The move is deliberately austere: to understand an object, don't open it up and inspect its elements — look instead at the arrows going into and out of it. It sounds like throwing away information. It isn't. We'll soon be able to pin down ideas like isomorphism, product, and even the empty set using nothing but arrows and how they compose.

§2 — The definition

A category is four pieces of data and two laws

Here it is in full. It is short. Resist the urge to find it underwhelming — its shortness is exactly what lets it apply everywhere.

Definition · Category

A category 𝒞 consists of:

  • a collection of objects — A, B, C, …
  • for each pair of objects X, Y, a collection of morphisms (arrows) from X to Y, written 𝒞(X, Y); an arrow is written f : X → Y
  • for every object X, an identity arrow idX : X → X
  • a composition rule: given f : X → Y and g : Y → Z, an arrow g ∘ f : X → Z

subject to two laws:

Associativityh ∘ (g ∘ f) = (h ∘ g) ∘ f
IdentityidY ∘ f = f = f ∘ idX

Three things to internalize right away.

Composition reads right-to-left. The expression g ∘ f means “first f, then g,” matching how g(f(x)) works. The arrow on the right goes first.

Arrows are typed. You can only compose g ∘ f when the target of f is the source of g. Most pairs of arrows simply cannot be composed, and that’s the point — the bookkeeping of sources and targets is doing real work.

That is the entire toolkit. Objects, typed arrows, who-composes-with-whom, and which arrows are identities. There is no notion of an element, no “inside” of an object. Whatever we want to say, we must say with arrows.

A small honest footnote

Why “collection” rather than “set”? Because the category of all sets has too many objects to itself be a set — assuming otherwise revives Russell’s paradox. When every 𝒞(X, Y) is a genuine set we call 𝒞 locally small; if the objects form a set too, small. You can safely file this away for now; it changes none of the reasoning below.

§3 — Diagrams & commuting

How category theorists write equations: they draw them

A diagram is just a picture of some objects and some arrows between them. A diagram commutes when any two directed paths sharing the same start and the same end compose to the same arrow. Commutativity is an equation — drawn rather than written.

§4 — A gallery of categories

Five examples that stretch the definition in five directions

The definition is abstract on purpose. The only cure for “but what is a category, really?” is to meet enough of them that the pattern, not any one instance, becomes the thing you see.

i. Set — the prototype

Objects are sets; an arrow X → Y is a function from X to Y; composition is ordinary function composition; idX is the function that returns its input. The laws hold for the most pedestrian reasons: composing functions is associative, and sandwiching with the identity function changes nothing. Keep Set in your pocket as the example to sanity-check every new idea against.

ii. A preorder is a category

Take any set with a relation ≤ that is reflexive and transitive. Build a category: the objects are the elements, and you draw exactly one arrow x → y whenever x ≤ y (and none otherwise). Reflexivity (x ≤ x) hands you every identity; transitivity (x ≤ y ≤ z ⟹ x ≤ z) hands you composition. The two laws come for free, because between any two objects there is at most one arrow, so any two arrows with matching ends are automatically equal.

iii. A monoid is a one-object category

A monoid is a set M with an associative operation and a unit e. Make a category with a single object •, let the arrows • → • be the elements of M, let composition be the monoid’s operation, and let id• = e. The category axioms become, line for line, the monoid axioms. Run it backwards and the same is true: any category with one object is a monoid.

iv. Tiny categories you can draw in full

Call 𝟙 the category with one object and nothing but its identity. Call 𝟚 the category with two objects and a single non-identity arrow 0 → 1. These look like toys, but they earn their keep: a structure-preserving map out of 𝟚 into some category 𝒞 is exactly a choice of one arrow in 𝒞. Small categories are the “shapes” you probe other categories with.

v. Types and programs

In a typed programming language, take the types as objects and (pure, total) functions as arrows; composition is function composition and the identity is lambda x: x. This is the category programmers already work inside without naming it — which is exactly why the next section, where we drop into Python, will feel less like translation and more like recognition.

§5 — Category theory in Python

Make the laws concrete: turn them into assertions

Abstraction earns trust faster when you can poke it. First, inside Set, functions really are morphisms — let’s build a composite by hand and watch the identity and associativity laws hold on an actual example.

set_as_category.py — morphisms are functions
def compose(g, f):
    "g after f:  (g . f)(x) = g(f(x))"
    return lambda x: g(f(x))

A = {1, 2, 3}
B = {'a', 'b'}
C = {True, False}

f = {1: 'a', 2: 'b', 3: 'a'}     # f : A -> B
g = {'a': True, 'b': False}      # g : B -> C

gf = {x: g[f[x]] for x in A}     # the composite g . f : A -> C
print(gf)

assert {x: f[x] for x in A} == f          # identity law:  f . id_A == f

h = {True: 1, False: 0}                    # h : C -> {0, 1}
left  = {x: h[g[f[x]]] for x in A}         # (h . g) . f
right = {x: h[g[f[x]]] for x in A}         #  h . (g . f)
assert left == right                       # associativity
print("laws ok")
output{1: True, 2: False, 3: True} laws ok

That checks one example in an infinite category. To do better, package the definition itself as data — objects, a typed list of arrows, a composition table, and a chosen identity per object — and write one function that verifies all four requirements at once. For a finite category, this is not a spot-check: it’s a complete proof, because the laws are quantified over a finite set of arrows.

finite_category.py — a law-checker
from itertools import product

class FiniteCategory:
    """objects, arrows (name -> (dom, cod)), a composition table,
       and a chosen identity arrow for each object."""
    def __init__(self, objects, arrows, comp, ident):
        self.obj, self.ar, self.comp, self.id = set(objects), arrows, comp, ident

    def dom(self, a): return self.ar[a][0]
    def cod(self, a): return self.ar[a][1]
    def composable(self, g, f): return self.dom(g) == self.cod(f)

    def check(self):
        for X in self.obj:                              # identities typed X -> X
            assert self.ar[self.id[X]] == (X, X)
        for g, f in product(self.ar, repeat=2):         # composition total & typed
            if self.composable(g, f):
                gf = self.comp[(g, f)]
                assert self.ar[gf] == (self.dom(f), self.cod(g))
        for f in self.ar:                               # unit laws
            X, Y = self.ar[f]
            assert self.comp[(self.id[Y], f)] == f
            assert self.comp[(f, self.id[X])] == f
        for h, g, f in product(self.ar, repeat=3):      # associativity
            if self.composable(h, g) and self.composable(g, f):
                assert self.comp[(h, self.comp[(g, f)])] == \
                       self.comp[(self.comp[(h, g)], f)]
        return True

# A monoid as a ONE-object category: integers mod 3 under addition.
arrows = {n: ('*', '*') for n in range(3)}            # the arrows ARE the elements
comp   = {(a, b): (a + b) % 3 for a in range(3) for b in range(3)}
Z3 = FiniteCategory({'*'}, arrows, comp, ident={'*': 0})
print(Z3.check())
outputTrue

The same check() validates the preorder of example (ii) and the category 𝟚 of example (iv) without a single change — you only feed it different data. Building 𝟛 from the chain 0 ≤ 1 ≤ 2 and running the checker on it is the last exercise below.

Going deeper

Special arrows: monomorphisms and epimorphisms

In Set the injective and surjective functions play starring roles — but “injective” and “surjective” are defined by talking about elements, which this course has forsworn. The arrows-only surrogates are cancellation properties, and they will follow us through the whole course (equalizers produce them in Lesson 5; adjunctions detect them in Lesson 6).

Definition · Mono & epi

An arrow m : X → Y is a monomorphism (monic) if it is left-cancellable: m ∘ h = m ∘ k implies h = k. Dually, e is an epimorphism (epic) if it is right-cancellable: h ∘ e = k ∘ e implies h = k.

In Set, monic is exactly injective and epic is exactly surjective — an injective function can be “undone from the left,” conceptually, so nothing precomposed with it can hide. But the translation is not perfect across categories, and the standard warning is the point of Exercise 11: mono + epi need not imply iso, even though “injective + surjective = bijective” in Set. The failure happens in the humblest categories we have — a lesson in how much of one’s intuition is secretly about sets.

§6 — Exercises

Twelve problems, from finger exercises to small proofs

Work each one before opening its solution — the arrows-only proofs especially, since the discipline of never mentioning an element is the skill this course is training. Several exercises reuse the FiniteCategory checker from §5, so keep a Python session open.

Exercise 1warm-up

Let A = {1, 2}, B = {'a', 'b'}, and C = {10, 20}, with functions given as dictionaries: f = {1:'a', 2:'b'} representing f : A → B, and g = {'a':10, 'b':20} representing g : B → C.

Write down the dictionary representing the composite g ∘ f : A → C. Then say, in one sentence, why the notation g ∘ f — with g written first — nevertheless means “apply f first.”

Show solution

Tracing each element: 1 ↦ 'a' ↦ 10 and 2 ↦ 'b' ↦ 20, so g ∘ f is the dictionary {1: 10, 2: 20}, or in code, {x: g[f[x]] for x in f}.

The order comes from function application: (g ∘ f)(x) is defined to be g(f(x)), and reading g(f(x)) inside-out, the innermost function — f — acts first. The arrow written last is applied first. This right-to-left convention trips up almost everyone at least once, which is why it is Exercise 1.

Exercise 2arrows-only

The definition of a category asks for an identity arrow on each object. This exercise shows the wording could have said the identity arrow.

Suppose i : X → X and i′ : X → X both behave as identities on X — that is, each satisfies the identity law: composing with either of them, on either side, changes nothing. Prove that i = i′, using nothing but the identity law itself.

Show solution

Consider the single composite i′ ∘ i and evaluate it twice. Because i′ acts as an identity (applied on the left), i′ ∘ i = i. Because i acts as an identity (applied on the right), i′ ∘ i = i′. One arrow, two names: i = i′.

Notice what the proof did not use: anything about what X is, or contains. It is pure arrow-pushing, and it licenses the notation idX — the identity on X — for the rest of the course.

Exercise 3structure

Lesson 1 claimed that a monoid is the same thing as a one-object category. A group is a monoid in which every element has an inverse.

State the full dictionary that translates “one-object category in which every arrow is invertible” into “group,” and back: say what the arrows, the composition, the identity arrow, and the invertibility condition each become on the group side, and check that the axioms match up.

Show solution

The single object carries no information — it is just a peg to hang arrows on. The arrows become the group’s elements; composition of arrows becomes the group operation; the identity arrow becomes the identity element e; and “every arrow is invertible” — for each f there is a g with g ∘ f = f ∘ g = id — becomes the existence of inverses.

The category axioms translate line for line: associativity of composition is associativity of the operation, and the identity law is the unit law. So: groups are exactly the one-object categories in which every arrow is invertible. Drop the invertibility requirement and the same dictionary yields monoids, as in example (iii) of §4.

Exercise 4examples

Each of the following proposes a category whose objects are all sets. For each, decide whether it really is a category, and if not, identify exactly which requirement fails.

  1. Arrows are the injective functions.
  2. Arrows are the surjective functions.
  3. Arrows are the functions that are not the identity.
  4. Arrows are the functions that have at least one fixed point (some x with f(x) = x).
Show solution

(a) Yes and (b) yes. A composite of injections is an injection, a composite of surjections is a surjection, and the identity function is both — each class is closed under composition and contains all the identities it needs. Associativity is inherited from ordinary function composition.

(c) No, twice over: it excludes the identity arrows by fiat, and it is not even closed under composition — a bijection composed with its inverse lands exactly on an identity, which the class forbids.

(d) No, and this one is sneakier: the identities are all present (every point of id is fixed), but closure fails. On {a, b, c}, let f swap a, b and fix c, and let g swap b, c and fix a. Each has a fixed point, but tracing the composite g ∘ f element by element gives the 3-cycle a ↦ c, b ↦ a, c ↦ b, which fixes nothing. The moral of (c) and (d): the closure conditions are not bureaucratic fine print; drop them and the structure genuinely collapses.

Exercise 5arrows-only

In §4, example (ii), we turned any preorder into a category by drawing exactly one arrow x → y whenever x ≤ y.

Explain why, in such a category, every diagram commutes — no matter how many objects and arrows it has, and without checking a single composite.

Show solution

A diagram commutes when any two directed paths with the same start and end compose to the same arrow. In a preorder-as-category there is at most one arrow between any ordered pair of objects. Two paths from x to y compose to two arrows x → y — and since at most one such arrow exists, they are forced to be equal. Commutativity is automatic.

Categories with this at-most-one-arrow property are called thin. They carry order information but no information about how one object maps to another — which is exactly why they make such clean test cases for definitions later in the course.

Exercise 6counting

Build the category 𝟛 from the chain 0 ≤ 1 ≤ 2, the same way 𝟚 was built from 0 ≤ 1.

How many arrows does 𝟛 have in total? List them. Then generalize: how many arrows does the chain 0 ≤ 1 ≤ ⋯ ≤ n have as a category?

Show solution

𝟛 has 6 arrows: the three identities on 0, 1, 2, the two generating arrows 0 → 1 and 1 → 2, and the composite 0 → 2. The last one is the arrow people forget — but transitivity of ≤ means it is there, and closure under composition obliges you to include it.

In general there is exactly one arrow i → j for each ordered pair with i ≤ j. Counting pairs: (n+1)(n+2)/2 arrows — the triangular numbers. For n = 2 this gives 6, as counted by hand.

Exercise 7structure · new

Every category has an evil twin. Given a category 𝒞, define its opposite category 𝒞op as follows: the objects are the same, but each arrow f : X → Y of 𝒞 is reinterpreted as an arrow f : Y → X of 𝒞op, and composition is defined by g ∘op f = f ∘ g, using the composition of 𝒞 on the right.

Verify that 𝒞op satisfies the two category laws, taking care with the order of composition. What is (𝒞op)op?

Show solution

Typing. If in 𝒞op we can form g ∘op f, then (unwinding the reversed directions) f ∘ g is composable in 𝒞, and its reinterpretation has exactly the right source and target — so the definition makes sense.

Identities. The same arrows idX serve: id ∘op f = f ∘ id = f and f ∘op id = id ∘ f = f, both by the identity law of 𝒞.

Associativity. h ∘op (g ∘op f) = (f ∘ g) ∘ h and (h ∘op g) ∘op f = f ∘ (g ∘ h); these agree by associativity in 𝒞.

Finally, reversing all arrows twice restores every arrow to its original direction, and the composition unwinds too: (𝒞op)op = 𝒞. Remember this construction — in Lesson 4 it becomes the duality principle, which lets every theorem be proved twice for the price of once.

Exercise 8arrows-only · new

An arrow f : X → Y is an isomorphism if there exists g : Y → X with g ∘ f = idX and f ∘ g = idY. Prove three basic facts, each by pure arrow-pushing:

  1. Every identity arrow is an isomorphism.
  2. The inverse of an isomorphism is unique (so we may write f−1).
  3. A composite of isomorphisms is an isomorphism, and (g ∘ f)−1 = f−1 ∘ g−1 — note the reversal of order.
Show solution

(a) idX is its own inverse: id ∘ id = id, twice over.

(b) Suppose g and g′ are both two-sided inverses of f. Then g = g ∘ id = g ∘ (f ∘ g′) = (g ∘ f) ∘ g′ = id ∘ g′ = g′. The middle step is associativity; everything else is the inverse and identity laws. This mirrors Exercise 2 — uniqueness proofs in category theory tend to have exactly this shape.

(c) Check that f−1 ∘ g−1 inverts g ∘ f: (f−1 ∘ g−1) ∘ (g ∘ f) = f−1 ∘ (g−1 ∘ g) ∘ f = f−1 ∘ id ∘ f = f−1 ∘ f = id, and symmetrically on the other side. The reversal of order is forced by the typing: to undo “first f, then g,” you must undo g first — socks and shoes.

Exercise 9code

Encode the category 𝟛 from Exercise 6 as a FiniteCategory — objects, all six arrows, the composition table, and the identities — and run .check() to confirm it satisfies all four requirements.

Hint: because 𝟛 is thin, there is at most one arrow between any two objects, so every entry of the composition table is forced; you can generate the table with a small helper rather than writing it out by hand.

Show solution

Name the three identities and the three non-trivial arrows, then let a helper fill the forced composition table:

three.py
objs = {0, 1, 2}
ar = {'i0':(0,0), 'i1':(1,1), 'i2':(2,2),
      'a':(0,1), 'b':(1,2), 'c':(0,2)}        # c is forced:  b . a
ident = {0:'i0', 1:'i1', 2:'i2'}

# at most one arrow per ordered pair, so each composite is determined
def only(s, t):
    return next(n for n,(d,c) in ar.items() if d==s and c==t)
comp = {(g,f): only(ar[f][0], ar[g][1])
        for g in ar for f in ar if ar[f][1]==ar[g][0]}

print(FiniteCategory(objs, ar, comp, ident).check())   # True
outputTrue

The comment on the arrow c is the real content: once you accept a : 0 → 1 and b : 1 → 2, the category has no choice but to contain their composite.

Exercise 10code · new

A checker is only trustworthy if you have seen it catch a mistake. Starting from your working 𝟛 of Exercise 9, corrupt the composition table on purpose: set comp[('b','a')] = 'i0', claiming that the composite of 0 → 1 followed by 1 → 2 is the identity on 0.

Before running it: which of the four checks in check() should fail, and why? Then run it and confirm.

Show solution

The typing check on composites fails first. The composite of a : 0 → 1 and b : 1 → 2 must be an arrow 0 → 2, but i0 is typed (0, 0) — so the assertion ar[gf] == (dom(f), cod(g)) raises AssertionError before the identity or associativity checks are even reached.

This is worth internalizing: most bookkeeping errors in category theory are typing errors, and they are caught mechanically, before any interesting law is consulted. It is the same discipline a type-checker imposes on a program — which is not a coincidence.

Exercise 11arrows-only · new

Using only the cancellation definitions from the Going-deeper section:

  1. Show every isomorphism is both monic and epic.
  2. Show that in a preorder-as-category every arrow is both monic and epic — and conclude, using an arrow of the chain 𝟚, that “mono and epi” does not imply “iso.”
Show solution

(a) Suppose m has inverse m−1 and m ∘ h = m ∘ k. Precompose both sides with m−1 and use associativity: h = (m−1 ∘ m) ∘ h = (m−1 ∘ m) ∘ k = k. Right-cancellation is the mirror computation with m−1 on the other side.

(b) In a thin category, any two parallel arrows h, k with the same source and target are already equal — so the implication “m ∘ h = m ∘ k ⟹ h = k” holds vacuously for every m, and likewise its dual. Now take the unique arrow u : 0 → 1 in 𝟚: it is monic and epic by the above, but it is not an isomorphism, since there is no arrow 1 → 0 at all. The moral: bijectivity is a theorem about Set, not a consequence of the axioms — cancellation and invertibility are genuinely different ideas that happen to coincide there.

Exercise 12structure · new

Given categories 𝒞 and 𝒟, define the product category 𝒞 × 𝒟: objects are pairs (C, D), arrows are pairs (f, g) of arrows, composed slotwise: (f′, g′) ∘ (f, g) = (f′ ∘ f, g′ ∘ g).

Say what the identity on (C, D) must be, and verify both category laws. (File this construction away — it is the stage on which Lesson 6 exhibits products and coproducts as adjoints.)

Show solution

The identity is (idC, idD), since composing slotwise with it applies each ordinary identity law in its own slot: (f, g) ∘ (id, id) = (f ∘ id, g ∘ id) = (f, g), and likewise on the left.

Associativity is the same story: ((f″, g″) ∘ (f′, g′)) ∘ (f, g) and (f″, g″) ∘ ((f′, g′) ∘ (f, g)) both compute, slot by slot, to (f″ ∘ f′ ∘ f,  g″ ∘ g′ ∘ g) — each slot protected by the associativity of its own category. Nothing interacts across slots, which is the whole design. Constructions like this, where the laws hold “componentwise for free,” recur constantly: the functor category of Lesson 3 inherited its laws pointwise in exactly the same way.

Recap

What to carry forward

  1. A category is objects + typed arrows + composition + identities, obeying associativity and the identity law. Nothing more.
  2. The arrows are the content. Whenever you’re tempted to reach inside an object, ask whether you can say the same thing with arrows instead — you usually can.
  3. Commutative diagrams are equations. A diagram commutes when same-endpoint paths compose equally.
  4. The definition is a wide net: Set, every preorder, every monoid, every group, and the world of typed programs are all categories.
  5. For a finite category the laws are checkable — and you wrote the checker.
