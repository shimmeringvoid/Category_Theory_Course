Intro to Category Theory · Lesson 02

Functors
Maps between categories

We just built categories. The reflex of the whole subject is to immediately ask for the right maps between them — and answering snaps categories themselves into a category.

§1 — The reflex

New objects? Ask for the maps

Lesson 1 handed us a new kind of mathematical object: the category. Category theory now does what it always does when it meets a new object — it asks for the right notion of map between them, the maps that respect whatever structure those objects carry.

A category’s structure is exactly four things: objects, arrows, identities, and composition. So a map of categories ought to send objects to objects and arrows to arrows, and leave identities and composition intact. That map is a functor, and pinning it down is the whole job of this lesson.

Functors are not exotic. “Build a list of,” “take the underlying set of,” “form the free group on,” “the fundamental group of a space” — every one of these is a functor. In code, every lawful map is a functor. They are how one region of mathematics gets translated into another, and the translation is faithful enough to carry whole theorems across the bridge.

§2 — The definition

Two assignments, two laws

The shape of the definition should feel familiar — it mirrors Lesson 1 line for line.

Definition · Functor

A functor F : 𝒞 → 𝒟 consists of:

  • an object map — to each object X of 𝒞, an object F X of 𝒟
  • an arrow map — to each arrow f : X → Y, an arrow F f : F X → F Y

satisfying:

IdentitiesF(idX) = idF X
CompositionF(g ∘ f) = F g ∘ F f

Mind the typing. The arrow map isn’t free to send f anywhere — F f must run from F X to F Y. A functor therefore transports an entire diagram in 𝒞 to a same-shaped diagram in 𝒟: whatever picture you draw in the source, F redraws it in the target with every dot and arrow relabelled. That is exactly what the triangle at the top of the page shows.

The laws are not arbitrary. They say: preserve the two things that made 𝒞 a category in the first place. Identities and composites are the structure; a functor is precisely a map of objects-and-arrows that keeps them.

Aside · Contravariance

Some natural constructions reverse arrows rather than preserving them. A contravariant functor sends f : X → Y to an arrow F f : F Y → F X and obeys F(g ∘ f) = F f ∘ F g. The everyday example: fix a set S and send each set X to the set of functions X → S; a map f : X → Y then induces “precompose with f” running the other way, (Y → S) → (X → S). The default kind is covariant; contravariant just flips every arrow. We’ll meet the systematic source of this — the opposite category — later.

§3 — What a functor guarantees

It can forget distinctions, but never invent them

Because a functor preserves composition and identities, it sends commuting diagrams to commuting diagrams: if two paths in 𝒞 are equal, their F-images are equal. A functor can never break an equation between composites — at worst it collapses things that were different, but it cannot pull apart things that were the same. From this single observation comes the first real theorem of the lesson.

Theorem · Functors preserve isomorphisms

If f : X → Y is an isomorphism — there is g : Y → X with g ∘ f = idX and f ∘ g = idY — then F f is an isomorphism too, with inverse F g.

Proof (arrows only). Apply F to g ∘ f = idX. Composition-preservation turns the left side into F g ∘ F f, and identity-preservation turns the right side into idF X, so F g ∘ F f = idF X. The symmetric computation gives F f ∘ F g = idF Y. Hence F g is a two-sided inverse of F f. ∎

Run that backwards and you get one of the most productive moves in mathematics. If some functor F sends X and Y to objects that are not isomorphic, then X and Y could not have been isomorphic either. The classic instance is the fundamental group π1 : Top∗ → Grp: a circle and a disk have different fundamental groups, so they are not the same space. Functors convert a hard question — are these spaces the same? — into an easier one about groups.

One caution, so the guarantee isn’t oversold: functors preserve isomorphisms but need not reflect them — F f being an iso does not force f to be one — and they need not preserve unrelated features like injectivity. The promise runs one direction only.

§4 — A gallery of functors

Six functors, and one of them is the subject eating its own tail

i. The identity functor, composition, and Cat

Every category 𝒞 has an identity functor id𝒞 fixing every object and arrow. And functors compose: given F : 𝒞 → 𝒟 and G : 𝒟 → ℰ, the assignment X ↦ G(F X), f ↦ G(F f) is again a functor — it preserves identities and composition because F and G each do. Functor composition is associative; identity functors are units. Read that back: it is the Lesson 1 definition of a category, with categories as objects and functors as arrows. That category is called Cat. We have climbed a floor — the thing we defined last lesson is itself an example of the thing we defined last lesson.

ii. Forgetful functors

The functor U : Grp → Set sends a group to its underlying set and a homomorphism to its underlying function — it simply forgets the multiplication. It qualifies as a functor for an almost embarrassing reason: a homomorphism already is a function, and composing or taking identities of homomorphisms is composing or taking identities of those functions. The same move gives Vect → Set, Top → Set, Ring → Ab, and dozens more. Forgetting structure is the most common functor in the wild.

iii. Free functors

Going the other way — building the freest possible structure on raw material — is where it gets interesting. Set → Mon sends a set A (an “alphabet”) to the free monoid on it: finite sequences of elements of A under concatenation, with the empty sequence as unit. That is exactly strings, exactly lists. A function between alphabets lifts to the map that rewrites each sequence letter by letter. Free and forgetful functors travel in matched pairs — an adjunction — which is the summit this course is climbing toward.

iv. Maps you already knew, in disguise

Recall from Lesson 1 that a one-object category is a monoid. A functor between two one-object categories must send the lone object to the lone object, and its arrow map is then a function on elements that preserves composition (the operation) and the identity (the unit) — i.e. a monoid homomorphism. Likewise a functor between two preorders-as-categories is forced to send the arrow witnessing x ≤ y to one witnessing F x ≤ F y, which is exactly a monotone function. Two familiar notions of “map” turn out to be the single notion “functor,” specialised.

v. Small categories as shapes

A functor 𝟙 → 𝒞 is a choice of an object of 𝒞 — the single object has to land somewhere. A functor 𝟚 → 𝒞 is a choice of an arrow: the non-identity arrow 0 → 1 must go to some arrow, and everything else is forced. This is the precise form of the teaser from Lesson 1. Tiny categories act as templates, and functors out of them pick that shape out inside any target — bigger shapes will pick out commuting squares, parallel pairs, and cones, which become the raw material of limits two lessons from now.

vi. The List functor, concretely

On the category of types, List sends a type A to List[A] and a function f : A → B to the function that applies f to every element. Its two functor laws — map(id) == id and map(g ∘ f) == map(g) ∘ map(f) — are exactly the laws every well-behaved map must obey, and they are why a compiler is permitted to fuse map(g) ∘ map(f) into a single pass. The next section makes all of this run.

§5 — Functors in Python

Two views: lists as a functor, and a checker for finite ones

First, List as a functor on types. We lift ordinary functions with fmap and watch the two laws hold on real data.

list_functor.py — map is a functor
def fmap(f, xs):
    "lift an ordinary function to act on a list, elementwise"
    return [f(x) for x in xs]

inc = lambda x: x + 1
sq  = lambda x: x * x
data = [1, 2, 3, 4]

# Functor law 1:  fmap(id) == id
assert fmap(lambda x: x, data) == data

# Functor law 2:  fmap(g . f) == fmap(g) . fmap(f)
assert fmap(lambda x: sq(inc(x)), data) == fmap(sq, fmap(inc, data))

print(fmap(inc, data))            # [2, 3, 4, 5]
output[2, 3, 4, 5]

Now the finite-category version. A functor between finite categories is just two dictionaries — where each object goes, where each arrow goes — plus a promise that typing, identities, and composites are respected. A single check() verifies all three (it reuses the FiniteCategory from Lesson 1). We point it at “multiply by 2” on the one-object category (ℤ/3, +) — which, because that category is a monoid, is asking whether doubling is a monoid homomorphism.

functor.py — a functor between finite categories
class Functor:
    """F : C -> D, given by a map on objects and a map on arrows."""
    def __init__(self, C, D, on_obj, on_arr):
        self.C, self.D, self.on_obj, self.on_arr = C, D, on_obj, on_arr

    def check(self):
        C, D, Fo, Fa = self.C, self.D, self.on_obj, self.on_arr
        for f in C.ar:                          # typing: F(f): F(dom) -> F(cod)
            X, Y = C.ar[f]
            assert D.ar[Fa[f]] == (Fo[X], Fo[Y])
        for X in C.obj:                         # preserves identities
            assert Fa[C.id[X]] == D.id[Fo[X]]
        for g, f in product(C.ar, repeat=2):    # preserves composition
            if C.composable(g, f):
                assert Fa[C.comp[(g, f)]] == D.comp[(Fa[g], Fa[f])]
        return True

# (Z/3, +) as a one-object category, carried over from Lesson 1
ar   = {n: ('*', '*') for n in range(3)}
comp = {(a, b): (a + b) % 3 for a in range(3) for b in range(3)}
Z3 = FiniteCategory({'*'}, ar, comp, {'*': 0})

# "multiply by 2" is a monoid homomorphism -- i.e. a functor of one-object cats
double = Functor(Z3, Z3, on_obj={'*': '*'},
                 on_arr={n: (2 * n) % 3 for n in range(3)})
print(double.check())
outputTrue

Feed the same check() the data for a functor 𝟚 → 𝒞 and you can watch “a functor out of 𝟚 is a choice of arrow” turn into a passing assertion for whichever arrow you pick — that’s the closing exercise.

Going deeper

Contravariance, formalized at last

§3’s aside mentioned functors that reverse arrows. With Lesson 1’s Exercise 7 in hand there is nothing new to define: a contravariant functor from 𝒞 to 𝒟 is simply an ordinary (covariant) functor 𝒞op → 𝒟. Reversal is not a new species of functor; it is the old species, fed the opposite category. The composition law comes out flipped — F(g ∘ f) = F(f) ∘ F(g) — because composition in 𝒞op is itself flipped.

The flagship example is a matched pair on Set. The covariant powerset functor P sends a set to its set of subsets and a function f to the direct image S ↦ f(S). The contravariant powerset functor sends f the other way, to the inverse image T ↦ f−1(T) — and the identity (g ∘ f)−1 = f−1 ∘ g−1, socks-and-shoes from Lesson 1’s Exercise 8, is exactly the flipped composition law. Contravariant Set-valued functors have a name of their own — presheaves — and the final lesson of this course is largely about one of them.

§6 — Exercises

Twelve problems on functors

As before, the arrows-only proofs are the heart of the set — real conclusions drawn entirely from preservation of composition and identities. The code problems reuse FiniteCategory and Functor from §5.

Exercise 1warm-up

With dbl = lambda x: 2*x, compute fmap(dbl, [1, 2, 3]) by hand.

Then state, precisely, the two laws that any well-behaved map function must satisfy, and say which functor law each corresponds to.

Show solution

fmap(dbl, [1,2,3]) = [2, 4, 6] — apply the function to each element, keep the structure.

The laws: map(id) == id (mapping the do-nothing function over a list changes nothing) and map(g ∘ f) == map(g) ∘ map(f) (mapping a composite in one pass equals mapping the pieces in two passes). These are precisely the functor laws — identity-preservation and composition-preservation — with List in the role of F. The second law is what licenses a compiler to fuse two traversals into one.

Exercise 2arrows-only

An arrow r is a left inverse (a retraction) of f : X → Y if r ∘ f = idX.

Show that any functor F sends a left inverse to a left inverse: if r ∘ f = idX, then F r ∘ F f = idF X. Where exactly does each functor law get used?

Show solution

Apply F to both sides of the equation r ∘ f = idX. On the left, composition-preservation gives F(r ∘ f) = F r ∘ F f. On the right, identity-preservation gives F(idX) = idF X. Equal arrows have equal images, so F r ∘ F f = idF X.

Each law is used exactly once, on its own side of the equation. Run the same argument on a right inverse and combine the two, and you have re-derived the §3 theorem that functors preserve isomorphisms — this exercise is its load-bearing half.

Exercise 3structure

Lesson 1 established that a one-object category is the same thing as a monoid. Now take two monoids M and N, viewed as one-object categories, and consider a functor F between them.

Spell out the full dictionary: what data does F carry, and what do “preserves composition” and “preserves identities” become, stated in the language of monoids?

Show solution

The object map is trivial — one object must go to one object — so the only genuine data is the arrow map, a function φ from the elements of M to the elements of N.

Preserves composition becomes φ(a · b) = φ(a) · φ(b): the function respects the monoid operation. Preserves identities becomes φ(eM) = eN: the unit goes to the unit. Those two conditions are, word for word, the definition of a monoid homomorphism. So functors between one-object categories and monoid homomorphisms are the same notion — the concept “functor” specializes correctly.

Exercise 4examples

Consider the forgetful functor U : Grp → Set, which strips a group down to its underlying set and a homomorphism down to its underlying function. Two questions:

  1. If U f = U g for two homomorphisms f, g between the same groups, must f = g?
  2. Is every function between the underlying sets of two groups of the form U h for some homomorphism h?

The two properties you are probing have names — faithful and full. Which does U have?

Show solution

(a) Yes. A homomorphism is completely determined by what it does to elements — its underlying function. Two homomorphisms with the same underlying function are the same homomorphism. So U is injective on each hom-set: it is faithful.

(b) No. A typical function between two groups’ underlying sets does not respect multiplication — e.g. any function sending the identity element somewhere other than the identity. Such functions are not U of anything, so U is not full.

“Faithful but not full” is the characteristic signature of forgetful functors: forgetting structure loses no arrows, but the target category has many arrows that never came from the source.

Exercise 5think

Nothing in the definition of a functor requires the object map to be injective or surjective. Make that vivid: exhibit one functor that is wildly non-injective on objects, and one that is badly non-surjective. Small categories from Lesson 1 (𝟙, 𝟚) are enough raw material.

Show solution

Non-injective: the unique functor 𝒞 → 𝟙 from any category with at least two objects. Every object is crushed onto the single object of 𝟙 and every arrow onto its identity — the laws hold trivially, and injectivity fails as badly as possible.

Non-surjective: an inclusion 𝟙 → 𝒞 that picks out one object of a large category 𝒞, missing every other object.

The first functor collapses; the second merely samples. Functors are free to do either — they promise coherence, not faithfulness to size or shape.

Exercise 6counting

Let Z3 denote the one-object category built from the monoid ({0, 1, 2}, + mod 3), as in §5.

  1. How many functors are there 𝟙 → Z3?
  2. How many functors are there 𝟚 → Z3?
  3. What does the answer to (b) generalize to, for functors 𝟚 → 𝒞 into an arbitrary category?
Show solution

(a) Exactly 1. The object of 𝟙 must go to ∗, and its identity arrow must go to the identity 0. Nothing is free.

(b) Exactly 3. Both identities of 𝟚 are forced to 0, but the arrow u : 0 → 1 may be sent to any of the three arrows 0, 1, 2 — and each choice automatically satisfies both functor laws, since there are no non-trivial composites in 𝟚 to constrain it.

(c) In general, functors 𝟚 → 𝒞 correspond exactly to arrows of 𝒞. That is the precise meaning of the slogan “𝟚 selects an arrow” — small categories are probes, and this one probes for arrows.

Exercise 7arrows-only · new

§4(i) asserted that functors compose. Prove it in full: given functors F : 𝒞 → 𝒟 and G : 𝒟 → ℰ, define G ∘ F by X ↦ G(F X) on objects and f ↦ G(F f) on arrows, and verify both functor laws. Note carefully which law of which functor you invoke at each step.

Show solution

Identities. (G ∘ F)(idX) = G(F(idX)) = G(idF X) = idG F X. The middle equality is F’s identity law; the last is G’s, applied at the object F X.

Composition. (G ∘ F)(g ∘ f) = G(F(g ∘ f)) = G(F g ∘ F f) = G(F g) ∘ G(F f). First F’s composition law, then G’s, applied to the arrows F g and F f.

Typing checks out at every stage, so G ∘ F is a functor. Together with the identity functors this is exactly what makes Cat a category — the claim §4 made and this exercise cashes in.

Exercise 8structure · new

Fix an object D of a category 𝒟. The constant functor ΔD : 𝒞 → 𝒟 sends every object of 𝒞 to D and every arrow of 𝒞 to idD.

Verify the two functor laws for ΔD. Then describe, in a sentence, what ΔD does to any commuting diagram in 𝒞.

Show solution

Identities: ΔD(idX) = idD = idΔDX — true by construction. Composition: ΔD(g ∘ f) = idD, while ΔD(g) ∘ ΔD(f) = idD ∘ idD = idD. Equal.

ΔD crushes any diagram — however elaborate — down to the single object D decorated with identity loops. It is the maximally forgetful functor: it remembers only that a diagram existed. Constant functors look like throwaways, but they are the yardstick against which limits and colimits are defined in Lesson 5 — a cone over a diagram will be exactly a natural transformation out of a ΔD.

Exercise 9code

Encode 𝟚 as a FiniteCategory and build the functor 𝟚 → Z3 that selects the arrow 2. Run check() to confirm it passes, then change the selected arrow to 0 and to 1 and confirm those pass too — matching the count of Exercise 6(b).

Show solution

The identities are forced to 0; only the image of u is a choice:

two_to_z3.py
# the category 2 = ( 0 --u--> 1 )
two_ar   = {'i0':(0,0), 'i1':(1,1), 'u':(0,1)}
two_comp = {('i0','i0'):'i0', ('i1','i1'):'i1',
            ('u','i0'):'u',  ('i1','u'):'u'}
Two = FiniteCategory({0,1}, two_ar, two_comp, {0:'i0', 1:'i1'})

for pick in (0, 1, 2):
    F = Functor(Two, Z3, on_obj={0:'*', 1:'*'},
                on_arr={'i0':0, 'i1':0, 'u':pick})
    print(pick, F.check())        # all three pass
output0 True 1 True 2 True

Three arrows in Z3, three functors out of 𝟚 — the correspondence of Exercise 6, witnessed by a loop.

Exercise 10code · new

Now watch the checker earn its keep. Define the map n ↦ (n + 1) mod 3 on the arrows of Z3 (with the object fixed) and feed it to Functor(...).check() as a claimed functor Z3 → Z3.

Before running it: which functor law does this map violate, and at what input? Verify both by hand and by running the checker.

Show solution

Both laws fail, and the identity law fails first in the checker’s order. The identity arrow of Z3 is 0, but the map sends 0 ↦ 1 ≠ 0 — identities are not preserved. Composition breaks too: F(a + b) = a + b + 1 while F(a) + F(b) = a + b + 2 (mod 3), which differ everywhere.

In monoid language, adding one is not a homomorphism of (ℤ/3, +) — it fails to fix the unit. The checker rejects it with an AssertionError on the identity clause. Contrast with §5’s “multiply by 2,” which fixes 0 and distributes over addition, and passes. The difference between the two maps is the functor laws.

Exercise 11structure · new

Verify the claims of the Going-deeper section:

  1. Show the direct-image assignment P(f)(S) = { f(x) : x ∈ S } satisfies both (covariant) functor laws.
  2. Show the inverse-image assignment f ↦ f−1 satisfies the identity law and the flipped composition law, and say precisely which Lesson 1 exercise the flip is.
Show solution

(a) Identity: the direct image of S under id is S itself. Composition: (g ∘ f)(S) = { g(f(x)) : x ∈ S } = g(f(S)) — take the image in two stages or one, same set. Both laws hold, so P is a functor Set → Set.

(b) Identity: id−1(T) = T. Composition: x ∈ (g ∘ f)−1(T) means g(f(x)) ∈ T, i.e. f(x) ∈ g−1(T), i.e. x ∈ f−1(g−1(T)) — so (g ∘ f)−1 = f−1 ∘ g−1, order reversed. That reversal is Lesson 1, Exercise 8(c) — the socks-and-shoes law for inverses — resurfacing as the composition law of a functor out of Setop. One phenomenon, two bookkeepings.

Exercise 12code · new

Check the covariant powerset functor’s two laws by brute force: over every subset S ⊆ {1, 2, 3}, verify pmap(id, S) == S and pmap(h1, pmap(h2, S)) == pmap(h1 ∘ h2, S) for a pair of test functions of your choice.

Show solution

Eight subsets, two laws each — sixteen assertions:

powerset.py
from itertools import combinations

def pmap(f, S):                       # direct image on subsets
    return {f(x) for x in S}

def subsets(S):
    S = list(S)
    return [set(c) for r in range(len(S)+1) for c in combinations(S, r)]

h1 = lambda x: x % 2
h2 = lambda x: x + 1
for S in subsets({1, 2, 3}):
    assert pmap(lambda x: x, S) == S
    assert pmap(h1, pmap(h2, S)) == pmap(lambda x: h1(h2(x)), S)
print("powerset functor laws: 16 instances pass")
outputpowerset functor laws: 16 instances pass

A functor whose “elements” are themselves sets — the first hint that Set-valued functors are data structures worth studying in their own right, which is where Lesson 7 ends up.

Recap

What to carry forward

  1. A functor maps objects to objects and arrows to arrows, preserving identities and composition — the two pieces of structure a category has.
  2. Functors send commuting diagrams to commuting diagrams and isomorphisms to isomorphisms; they may forget distinctions but never manufacture them.
  3. That iso-preservation is the licence behind algebraic invariants: differ after F, and you differed before.
  4. Identity functors plus functor composition make categories into a category, Cat — the subject becomes self-referential.
  5. Monoid homomorphisms and monotone maps are functors in disguise; in code, a lawful map is a functor.
